Day Five of Class:
Refactoring the Layout

By Jared Knueven,
future Junior Developer from The Iron Yard.

Welcome to the continuation of Day Two's assignment, where I get to go back in and refactor all of my layout in bootstrap. My first step was to go in and rework my code to see how I could manipulate the old to force new changes. Upon fiddling around with the old line, I came to the conclusion that working the old content and text copy into new code was the most efficient use of refactoring. What you see here, in this version of the Day Two Assignment, is the old layout using bootstrap, rather than HTML and CSS. Visual changes include new pictures to fit the revitalized blog format, along with updating the Font Awesome images for style. The organization got an overhaul with the grid formating of bootstrap into a column and row alignment, which includes nesting of certain columns.

I definitely like the ability to have more control over the layout that comes with Bootstrap's Gird. I especially like the capability to mold content size within Columns while still keeping media resizing in mind. The column sizing and media relation might take some getting used to, but the rule for its use seems simple enough. The end product should look like a coherent two column "blog-esque" website, which is navicable to every user without frustration on every media level.
